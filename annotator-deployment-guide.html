Optimal Configuration:

  • Quality Controller: 1 additional annotator for inter-rater reliability

Rationale:

  • Target: At least 200 validated annotations per annotator (1,000+ total)

  • Overlap for reliability testing: 10-15% of passages

üìã Annotator Instructions Package

Send to Each Annotator:
  • annotation-guide.html - Complete annotation guidelines
  • morrison-biblical-annotation-tool.html - The annotation interface
  • Assigned detector results files - CSV/JSON for their novels
  • Unique Annotator ID - e.g., "UIUC_ANN_001"
  • Contact information - For questions and support
Important Setup Notes:
  • Ensure all files are in the same directory for proper linking
  • Test the tool with sample data before distribution
  • Provide clear file naming conventions for submissions
  • Set up regular backup of annotation data

üìä Quality Assurance Plan

Inter-Rater Reliability Testing
  • Have 2-3 annotators work on the same 50 passages
  • Calculate Cohen's kappa for agreement
  • Target: Œ∫ > 0.70 for acceptable reliability
  • Resolve disagreements through discussion
Expert Validation
  • Expert reviews 10% of all annotations
  • Focus on high-confidence and edge cases
  • Provide feedback for annotator improvement
  • Final approval of annotation guidelines
